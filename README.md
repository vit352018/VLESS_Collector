# 🔄 VLESS Collector

Бот, который каждый час автоматически:
- Ищет бесплатные VPN-серверы в интернете
- Проверяет какие из них реально работают
- Сохраняет рабочие в файлы (чтобы можно было добавить в Karing / Hiddify и т.д.)
- Загружает файлы на **Яндекс Диск** по WebDAV
- Показывает красивую страницу статистики

Всё это работает **бесплатно** на серверах GitHub — твой компьютер не нужен.

---

- Аккаунт на [Яндексе](https://yandex.ru) (если хочешь загрузку на Яндекс Диск)

## 🚀 Установка — шаг за шагом

### Шаг 1. Скопировать репозиторий к себе

1. Открой страницу этого репозитория на GitHub
2. Нажми кнопку **Fork** (в правом верхнем углу)
3. Нажми **Create fork**

> 💡 Fork — это как "сделать свою копию". Теперь у тебя есть собственный репозиторий с этим ботом.

---

### Шаг 2. Включить автозапуск

1. В своём репозитории нажми вкладку **Actions** (вверху)
2. Если видишь жёлтое предупреждение — нажми **"I understand my workflows, enable them"**

> 💡 GitHub по умолчанию выключает автозапуск в форках. Это мы его включаем.

---

### Шаг 3. Добавить пароль от Яндекс Диска (необязательно)

Если хочешь, чтобы файлы автоматически копировались на Яндекс Диск:

**3а. Получи пароль приложения (не основной пароль!):**

1. Зайди на [id.yandex.ru/security/app-passwords](https://id.yandex.ru/security/app-passwords)
2. Нажми **«Создать пароль приложения»**
3. Тип — выбери **«Другой»**, название — напиши **«vless-bot»**
4. Нажми **«Создать»**
5. Скопируй пароль — он показывается **только один раз!**

> ⚠️ Это специальный одноразовый пароль только для этого бота.
> Основной пароль от Яндекса нигде не вводи.

**3б. Добавь данные в GitHub Secrets:**

1. В своём репозитории: **Settings** → **Secrets and variables** → **Actions**
2. Нажми **New repository secret**
3. Создай первый секрет:
   - Name: `YANDEX_LOGIN`
   - Secret: `твой_логин@yandex.ru`
   - Нажми **Add secret**
4. Создай второй секрет:
   - Name: `YANDEX_PASS`
   - Secret: `пароль_из_шага_3а`
   - Нажми **Add secret**

> 💡 Secrets — это зашифрованные переменные. GitHub хранит их безопасно
> и подставляет при запуске бота. Никто кроме тебя их не видит.

---

### Шаг 4. Сделать папку на Яндекс Диске публичной

Чтобы файлы можно было добавить как подписку в Karing:

1. Открой [disk.yandex.ru](https://disk.yandex.ru)
2. После первого запуска бота появится папка **vless-collector**
3. Нажми на неё правой кнопкой → **«Поделиться»**
4. Включи **«Публичный доступ»**
5. Скопируй ссылку — она понадобится в Karing

---

### Шаг 5. Первый запуск

1. Вкладка **Actions** → слева нажми **"Collect & Test VPN Configs"**
2. Справа нажми **"Run workflow"** → **"Run workflow"** (зелёная кнопка)
3. Подожди 10–15 минут

После завершения в папке `output/` репозитория появятся файлы.

## 📁 Какие файлы создаёт бот

| Файл | Что внутри |
|------|------------|
| `VLESS_WORKING.txt` | Все рабочие серверы (все протоколы) |
| `VLESS_ONLY.txt` | Только VLESS-серверы |
| `VMESS_ONLY.txt` | Только VMess-серверы |
| `TROJAN_ONLY.txt` | Только Trojan-серверы |
| `HYSTERIA_ONLY.txt` | Только Hysteria2-серверы |
| `TOP50.txt` | Топ-50 самых быстрых серверов |
| `stats.json` | Статистика в машиночитаемом формате |
| `index.html` | Красивая страница статистики |

3. Branch: **main**, папка: **/ (root)** — нажми Save

   `https://ТВО_ИМЯ.github.io/vless-collector/`

## 💻 Локальный запуск (с компьютера)

Если хочешь запустить не через GitHub, а прямо у себя:

```bash
# 1. Скачай репозиторий
git clone https://github.com/ТВО_ИМЯ/vless-collector
cd vless-collector

# 2. Установи библиотеки
pip install -r requirements.txt

# 3. Создай файл .env с данными Яндекса
cp .env.example .env
# Открой .env в блокноте и впиши свои данные

# 4. Запусти
python run.py
```

**Другие команды:**
```bash
python run.py --sources   # только скачать ключи (без теста)
python run.py --test      # только протестировать скачанные
python run.py --upload    # только загрузить на Яндекс Диск
python run.py --stats     # показать статистику в консоли
```

## ❓ Частые вопросы

**Бот не запускается — ошибка в Actions?**
Открой упавший запуск → нажми на шаг с ошибкой → посмотри текст ошибки.
Чаще всего причина: источники временно недоступны. Подожди час — запустится сам.

**Файлы не загружаются на Яндекс Диск?**
Проверь: правильно ли введён логин (с `@yandex.ru`) и пароль **приложения** (не основной).

**Серверов очень мало (меньше 10)?**
Это нормально если все источники временно недоступны. Следующий час их будет больше.

**Можно изменить частоту обновления?**
Да, в файле `.github/workflows/collect.yml` строка `cron: "0 * * * *"`:
- `"0 * * * *"` — каждый час
- `"0 */2 * * *"` — каждые 2 часа
- `"0 */6 * * *"` — каждые 6 часов

**Можно добавить свои источники?**
Да — открой `src/collector.py`, найди список `SOURCES` и добавь свою ссылку по образцу.

## 📂 Структура проекта

```
vless-collector/
├── src/
│   ├── main.py            ← главный файл, запускает всё по порядку
│   ├── collector.py       ← скачивает ключи с GitHub-репозиториев
│   ├── tg_scraper.py      ← парсит Telegram-каналы
│   ├── tester.py          ← проверяет серверы (TCP + TLS)
│   ├── geoip.py           ← определяет страну каждого сервера
│   ├── writer.py          ← записывает файлы по протоколам
│   ├── html_gen.py        ← генерирует HTML-страницу статистики
│   └── yandex_upload.py   ← загружает файлы на Яндекс Диск
├── .github/workflows/
│   ├── collect.yml        ← запускает бота каждый час
│   └── pages.yml          ← публикует страницу статистики
├── output/                ← сюда сохраняются готовые файлы
├── run.py                 ← для запуска с компьютера
├── requirements.txt       ← список библиотек Python
├── .env.example           ← шаблон файла с паролями
└── README.md              ← это руководство
```
